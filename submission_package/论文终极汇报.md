# 论文终极汇报

**论文标题：** Topology-Aware GPCR--G Protein Coupling Prediction via ESM-2 Embeddings and Curated Transmembrane Annotations

**基于拓扑感知的GPCR-G蛋白偶联预测：融合ESM-2嵌入与精选跨膜注释**

---

## 第一部分：研究背景与动机

### 1.1 生物学背景

G蛋白偶联受体（GPCR）是人类基因组中最大的膜蛋白家族，约占已批准药物靶点的34%[^1]。GPCR通过与细胞内的异三聚体G蛋白（分为Gq、Gi、Gs、G12/13四大家族）偶联，激活不同的下游信号通路。预测一个GPCR与哪个G蛋白家族偶联，是药物设计和信号转导研究中的核心问题[^2]。

结构生物学已经明确：Class-A GPCR主要通过其**胞内环**（intracellular loops, ICLs）与G蛋白相互作用，尤其是**第三胞内环（ICL3）**和**第二胞内环（ICL2）**构成了主要的对接界面[^3][^4]。

> **[生物学知识出处]**
> - ICL2/ICL3构成G蛋白主要对接界面：Flock et al., "Universal allosteric mechanism for Gα activation by GPCRs", *Nature* 524, 173-179 (2015)[^3]。该研究通过对所有已知GPCR-G蛋白复合物结构进行系统性分析，发现了保守的ICL2/ICL3-G蛋白界面接触模式。
> - GPCR-G蛋白结合的结构基础：Ma et al., "Structural basis for the binding of the G protein-coupled receptor rhodopsin to its G protein", *Science* 377, eabn0080 (2022)[^4]。该研究解析了视紫红质-Gi复合物的冷冻电镜结构，直接显示ICL2和ICL3深入G蛋白alpha亚基的空腔。
> - 早期发现ICL3在G蛋白选择性中的关键作用：Möller et al., "Prediction of the coupling specificity of G protein-coupled receptors to their G proteins", *Bioinformatics* 17(Suppl 1), S174-S181 (2001)[^12]。该工作首次用计算方法证明胞内环区域序列模式能预测偶联特异性。

### 1.2 现有方法的两个核心缺陷

| 缺陷 | 具体表现 | 后果 |
|------|----------|------|
| **单蛋白分类退化** | 多数方法将任务简化为"给GPCR分类"，所有样本共用一个固定的G蛋白模板（如人源GNAQ） | 模型无法学习配对交互规则，退化为GPCR-only分类器 |
| **缺少拓扑引导** | 使用全序列mean-pooled embedding，不区分结合区与非结合区 | 模型注意力漂移到N-tail/C-tail等非功能区域，忽略真正的ICL2/3结合界面 |

### 1.3 本研究的解决策略

本研究提出三项改进：

1. **真正的配对任务构建**——每条样本是一个(GPCR, G蛋白家族)对，同一GPCR可出现在多行但配不同家族，迫使模型利用G蛋白侧信息
2. **拓扑感知特征工程**——通过UniProt精选跨膜注释精确提取ICL2/3边界，注入结构先验知识
3. **严格的同源性控制评估**——设计三级递进的交叉验证方案（Random CV → Cluster-aware CV → LOGPSO），杜绝序列泄露

## 第二部分：数据与方法

### 2.1 数据集构建

**数据来源：**
- **GPCRdb/IUPHAR**：从GPCRdb官方数据仓库下载的IUPHAR偶联注释，共1,596对[^7]
- **Local seed**：项目初始阶段收集的100个GPCR的Gq偶联标签，共51对

**原始数据文件：**
- IUPHAR偶联原始CSV：`paired_dataset/iuphar_coupling_data.csv`（399条GPCR记录）
- 原始配对矩阵：`paired_dataset/pairing_matrix_raw.csv`（1,647行，合并去重前）

**配对矩阵生成逻辑：**
1. 从IUPHAR CSV解析每条GPCR的PrimaryTransducer/SecondaryTransducer字段
2. 对每个GPCR生成4行（分别对应Gq/Gi/Gs/G12_13），有coupling evidence的标为1，否则标为0
3. 与Local seed合并，冲突时优先GPCRdb注释
4. 去除完全重复的(gpcr_id, g_protein_family)行

**重要说明——负样本定义：** IUPHAR数据中的负标签（coupling=0）表示"在该GPCR的已知偶联列表中未出现该G蛋白家族"。这属于**absence of evidence**而非**evidence of absence**——部分标为0的配对可能在未来的实验中被证实为正样本。这会系统性压低模型的表观precision。

#### 最终数据集统计

| 指标 | 数值 |
|------|------|
| 原始行数（合并前） | 1,647 |
| 去除缺失ESM-2特征（isoform P30988-2, P34998-2） | -4 |
| 去重后的另外重复 | -4 |
| **最终可评估配对数** | **1,639** |
| 唯一GPCR数 | 448 |
| 同源性聚类数（30%阈值） | 387 |
| 正样本（确认偶联） | 417（25.4%） |
| 负样本 | 1,222（74.6%） |
| 各家族对数（Gq / Gi / Gs / G12/13） | 448 / 397 / 397 / 397 |

> **Gq对数偏多的原因：** Local seed的51对全部是Gq家族的标签（项目初始阶段专注于Gq偶联研究），这些额外的51对导致Gq家族有448对而其他三个家族各397对。

### 2.2 特征提取流程

#### 2.2.1 ESM-2嵌入

主实验使用ESM-2 (`esm2_t6_8M_UR50D`, 320维)[^5][^6]提取残基级embedding，mean pooling得到序列级表征。为验证模型规模对结论的影响，同步提取并测试了更大规模的 `esm2_t33_650M_UR50D` (1280维)。

> **[ESM-2模型出处]**
> - Rives et al., "Biological structure and function emerge from scaling unsupervised learning to 250 million protein sequences", *PNAS* 118, e2016239118 (2021)[^5]
> - Lin et al., "Evolutionary-scale prediction of atomic-level protein structure with a language model", *Science* 379, 1123-1130 (2023)[^6]

对GPCR和G蛋白分别提取：
- GPCR：对每条GPCR序列提取mean-pooled embedding（主实验320-d，对照实验1280-d）
- G蛋白：对每个代表性亚型序列提取mean-pooled embedding，然后按家族映射（GNAQ→Gq, GNAI1/2/3→Gi, GNAS→Gs, GNA12/13→G12_13）

G蛋白代表序列存储在：`paired_dataset/g_protein_esm_features.json`

#### 2.2.2 UniProt跨膜拓扑注释

**注释来源：** UniProt REST API[^8]的`ft_transmem`字段，返回实验验证或计算预测的跨膜螺旋区域坐标。

**原始数据文件：** `paired_dataset/uniprot_topology.json`（346KB，包含420条GPCR的跨膜区域注释）

**注释示例（Q9UHM6，人源Melanopsin/OPN4）：**
```json
{
  "tm_regions": [[73,93], [108,128], [145,165], [189,209], [239,259], [297,317], [333,353]],
  "loops": {
    "N-tail": [1, 72],
    "ICL1": [94, 107],
    "ECL1": [129, 144],
    "ICL2": [166, 188],
    "ECL2": [210, 238],
    "ICL3": [260, 296],
    "ECL3": [318, 332],
    "C-tail": [354, 478]
  }
}
```

**覆盖率：** 448个GPCR中431个可查询UniProt accession（17个为isoform或无稳定accession），其中420个有跨膜注释，415个具有完整的7-TM螺旋结构（415/420 = 98.8%）。不足11条序列回退到Kyte-Doolittle滑窗法预测。

#### 2.2.3 从7-TM结构到胞内环

Class-A GPCR是一种七次跨膜蛋白，相邻跨膜螺旋之间的连接片段称为**loop**：
- **ICL2** = TM3末端+1 到 TM4起始-1
- **ICL3** = TM5末端+1 到 TM6起始-1

ICL2和ICL3是GPCR与G蛋白alpha亚基物理接触的主要界面[^3][^4]。

**提取流程：** 读取UniProt 7-TM区域坐标后，计算ICL2/3序列片段，再分别提取8项理化统计 + mean-pooled ESM-2局部embedding。

#### 2.2.4 ICL2/3的8项理化统计

| 特征 | 含义 | 生物学意义 |
|------|------|------------|
| length | 残基数 | ICL3长度变异性大，直接影响构象灵活性[^3] |
| mean_hydro | 平均疏水性（Kyte-Doolittle标度）[^9] | 反映loop的亲水/疏水倾向 |
| std_hydro | 疏水性标准差 | 反映氨基酸组成的异质性 |
| net_charge | 净电荷 | 与Gα表面互补性相关[^4] |
| pos_charge_ratio | 正电荷残基(K,R,H)占比 | -- |
| neg_charge_ratio | 负电荷残基(D,E)占比 | -- |
| hydrophobic_ratio | 疏水残基占比 | 参与疏水核心packing |
| aromatic_ratio | 芳香族残基(F,W,Y)占比 | 芳香残基参与π-堆叠和信号传导[^4] |

### 2.3 特征消融配置（8M / 650M 维度对照）

| 配置名称 | 组成 | 8M维度 | 650M维度* |
|----------|------|--------|-----------|
| Baseline | GPCR全局ESM-2 + G蛋白全局ESM-2 | 640-d | 1,600-d |
| + ICL stats | Baseline + ICL2/3理化统计 | 656-d | 1,616-d |
| + ICL full | Baseline + ICL2/3局部ESM-2 + ICL2/3理化统计 | 1,296-d | 4,176-d |
| ICL only | ICL2/3局部ESM-2 + ICL2/3理化统计（不含全局embedding） | 656-d | 2,576-d |

\* G蛋白特征仍使用320-d（统一提取自8M模型），因此650M Baseline = 1280 + 320 = 1600-d。

### 2.4 模型选择

使用SVM（RBF核，C=10.0，class_weight='balanced'，概率输出），标准化预处理。刻意选择简单模型是为了**隔离特征工程的效果**——排除模型架构差异带来的混杂因素。

### 2.5 三级递进评估方案

| 评估策略 | 分折依据 | 意义 |
|----------|----------|------|
| **Random CV** | 随机分层5折 | 乐观上界，允许同源序列出现在训练和测试集 |
| **Cluster-aware CV** | 按同源聚类分折（CD-HIT 30%阈值），同一GPCR的所有配对在同一折 | 现实估计，模型必须泛化到完全未见过的受体 |
| **LOGPSO** | 留出整个G蛋白家族 | 极端测试，评估跨家族迁移能力 |

> **[同源聚类方法出处]** CD-HIT: Li & Godzik, "Cd-hit: a fast program for clustering and comparing large sets of protein or nucleotide sequences", *Bioinformatics* 22(13), 1658-1659 (2006)[^10]。

## 第三部分：核心实验结果

### 3.1 特征消融实验（主表）

**V1 结果（原始特征，8M ESM-2）：**

| 特征配置 | 维度 | Random CV AUC | Cluster CV AUC | LOGPSO AUC |
|----------|------|---------------|----------------|------------|
| Baseline | 640 | 0.815 +/- 0.031 | 0.797 +/- 0.022 | 0.638 |
| + ICL stats | 656 | 0.832 +/- 0.017 | 0.810 +/- 0.030 | 0.628 |
| + ICL full | 1,296 | 0.839 +/- 0.014 | **0.830 +/- 0.041** | 0.609 |
| ICL only | 656 | 0.598 +/- 0.025 | 0.683 +/- 0.035 | 0.551 |

**V2 结果（加入家族先验 + ICL相似度，8M ESM-2）：**

| 特征配置 | 维度 | Random CV AUC | Cluster CV AUC | LOGPSO AUC |
|----------|------|---------------|----------------|------------|
| Baseline | 640 | 0.815 +/- 0.031 | 0.798 +/- 0.014 | 0.638 |
| + ICL stats V2 | 658 | **0.848 +/- 0.020** | **0.836 +/- 0.019** | 0.631 |
| + ICL full V2 | 1,298 | 0.848 +/- 0.015 | **0.842 +/- 0.005** | 0.606 |

**V2 结果（加入家族先验 + ICL相似度，650M ESM-2）：**

| 特征配置 | 维度 | Random CV AUC | Cluster CV AUC | LOGPSO AUC |
|----------|------|---------------|----------------|------------|
| Baseline | 1,600 | 0.820 +/- 0.033 | 0.819 +/- 0.017 | 0.606 |
| + ICL stats V2 | 1,618 | 0.830 +/- 0.032 | **0.833 +/- 0.014** | 0.601 |
| + ICL full V2 | 4,178 | 0.822 +/- 0.028 | 0.832 +/- 0.019 | 0.593 |

**关键发现：**

- 拓扑增强将Cluster CV AUC从**0.797提升至0.830**（V1 8M, +0.033）
- 加入家族级Gq正样本率和ICL相似度后，8M模型进一步提升至**0.842**（+0.045），650M模型提升至**0.833**（+0.014）
- **模型规模与特征增益的非单调关系**：650M Baseline 已高达 0.819，但 V2 增强后的相对增益（+0.014）小于 8M（+0.045），说明更大模型的embedding本身已编码部分家族/物种信息，对显式先验的边际响应减弱
- ICL-only模型Random CV仅0.598，说明全局ESM-2上下文不可替代——局部拓扑特征和全局序列表征是互补关系而非替代关系
- Random CV到Cluster CV的AUC下降（0.815→0.797）反映了同源泄露效应的存在

### 3.2 LOGPSO跨家族泛化分析

**LOGPSO各家族AUC（Baseline模型）：**

| 留出家族 | G12/13 | Gi | Gq | Gs |
|----------|--------|-----|-----|-----|
| AUC | 0.633 | 0.643 | 0.662 | 0.614 |

**核心发现：** 所有含全局G蛋白embedding的模型在LOGPSO下AUC仅0.60左右，且精确率/召回率/F1均接近0。

**根本原因：** 这是**评估协议与表征方式的结构性不兼容**，而非生物学瓶颈。具体机制如下：

```
训练阶段见过: Gq向量(320-d), Gi向量(320-d), Gs向量(320-d)
测试阶段遇到: G12/13向量(320-d) ← 从未见过

因为每个G蛋白家族只有一个固定320维向量，
所以测试集所有样本共享同一个未见过的常量块
→ RBF核距离被这个常量偏移主导
→ 所有测试样本获得近乎相同的预测概率（sigma ~ 10^-8）
→ 无法区分正负样本
```

**关键区分：** 这不意味着concatenated embedding本身有缺陷——在within-distribution任务（Random CV/Cluster CV）中它工作良好。LOGPSO暴露的是"该协议"与"该表征类型"的组合无法工作，任何使用固定向量编码G蛋白的方法在此设置下都必然失败。

**ICL-only模型的验证：** 由于不包含G蛋白embedding，避免了常量偏移问题，在LOGPSO下确实能产生非零分类指标（如Gi：precision=0.41, recall=0.92），间接证实了退化源于G蛋白固定向量表征。

### 3.3 可解释性分析

#### 3.3.1 Permutation Importance（特征重要性排序）

| 排名 | 特征 | AUC下降 | 对比参考 |
|------|------|---------|----------|
| 1 | ICL2 aromatic_ratio | 0.00434 | > ESM-2单维最大值(0.00310) |
| 2 | ICL3 length | 0.00417 | > ESM-2单维最大值(0.00310) |
| 3 | ICL3 net_charge | 0.00295 | -- |
| 4 | ICL3 std_hydro | 0.00226 | -- |
| 参考 | ESM-2 GPCR 320维均值 | 0.00068 | -- |
| 参考 | ESM-2 GPCR 320维中位数 | 0.00040 | -- |
| 参考 | ESM-2 GPCR 320维最大值 | 0.00310 | -- |

**生物学意义：**
- **ICL2芳香族比例**排名第一：芳香族残基（F, W, Y）在GPCR-G蛋白界面中已知参与疏水堆叠和信号传导关键相互作用[^4]
- **ICL3长度**排名第二：ICL3长度变异性大（从10到超过200个残基），长度直接影响G蛋白对接构象的灵活性[^3]
- **ICL3净电荷**排名第三：与G蛋白alpha亚基的带电表面互补性直接相关[^4]

#### 3.3.2 SHAP区域注意力分析

| 序列区域 | Baseline模型权重 | 拓扑增强后权重 | 变化 |
|----------|-----------------|---------------|------|
| N-tail | 16.9% | -- | 下降 |
| C-tail | 20.2% | -- | 下降 |
| ICL2 | 3.1% | 12.4% | +9.3% |
| ICL3 | 0.5% | 9.7% | +9.2% |

**解读：** 加入ICL拓扑特征后，模型权重从非结合区域（N-tail/C-tail）向已知结合界面（ICL2/ICL3）重新分配。这与"拓扑注释引导模型关注生物学相关区域"的假设一致。需要注意的是，权重转移可能部分来源于新增ICL特征本身的预测能力，而非模型内部表征的因果性修正。

### 4.4 关键洞察——模型规模的临界差异

**假阳性大幅下降（8M & 650M）：**
蜜蜂和蝴蝶opsin的预测概率从V1的0.55-0.74降至V2的0.31-0.44（8M）和0.40-0.46（650M）。这归功于**家族级Gq正样本率先验**——训练集中opsin家族（7条）只有OPN4一个Gq正样本（正样本率14.3%）。

**8M vs 650M的截然不同表现：**
- **8M V2**：鱼melanopsin（AAO38857.1）和蜘蛛kumopsin1（BAG14330.1）被家族先验压到阈值以下，产生**2个假阴性**。干净样本准确率71.4%。
- **650M V2**：1280-d的 richer embedding 抵抗了家族先验的压制，**2个远缘正样本全部正确**（鱼0.852，蜘蛛0.544），但新增1个假阴性（鱿鱼opsin 0.519）。干净样本准确率跃升至**85.7%**。

**核心洞察：** 650M模型更丰富的embedding能够**抵抗家族先验对远缘正样本的压制**，带来显著的跨物种泛化提升。

### 4.5 标签可靠性问题

测试集中5个"No"标签里有4个（蜜蜂×3、蝴蝶×1）是基于"该物种使用非人源Gq"推断的，这在逻辑上**不能推出**"不能激活人源GNAQ"。在人源GNAQ框架下，这些标签属于**未经实验验证的空白区域**（D级不可靠）。

**这意味着：** V1中的一部分"错误"预测可能实际上是基准标签本身的缺陷，而非模型泛化能力差。最务实的处理方式是：对于没有直接"在人源GNAQ上测试过"证据的样本，应标注为**UNKNOWN**并从定量评估中移除。

## 第五部分：三个核心结论

### 结论一：配对任务框架消除了单蛋白退化
将任务从"GPCR分类"重构为"(GPCR, G蛋白家族)配对预测"后，SHAP分析确认所有320个G蛋白ESM-2维度均有非零贡献，验证了模型确实在学习配对交互信息，而非退化为GPCR-only分类器。

### 结论二：拓扑注释将预训练embedding引向生物学合理的决策边界
UniProt精选跨膜注释提供了Kyte-Doolittle启发式方法无法提供的精准结构引导：
- 仅16维理化统计即可带来可观增益
- ICL3长度、ICL2芳香族比例等特征与结构生物学证据高度吻合[^3][^4]
- 模型权重从非功能区重新分配到已知结合界面

### 结论三：跨家族泛化需要新型交互表征
LOGPSO实验揭示了concatenated fixed-vector表征在leave-one-family-out场景下的结构性局限：
- 这不是生物学瓶颈，而是**协议-表征不兼容**
- 未来方向：残基级交叉注意力、attention-based co-embedding、AlphaFold界面接触预测

## 第六部分：方法创新点总结

| 创新维度 | 现有方法 | 本研究 |
|----------|----------|--------|
| 任务构建 | 单蛋白分类（固定G蛋白模板） | 真正的(GPCR, G蛋白家族)配对预测 |
| 拓扑信息 | 无，或使用启发式滑窗法 | UniProt精选7-TM注释 + 精准ICL边界提取 |
| 特征设计 | 全局mean-pooled embedding | 全局 + 局部ICL embedding + 理化统计 + **家族级偶联先验 + ICL同源相似度** |
| 评估严格度 | 通常仅Random CV | 三级递进：Random → Cluster-aware → LOGPSO |
| 可解释性 | 通常缺失 | Permutation importance + SHAP区域映射 |
| 失败模式分析 | 通常不讨论 | LOGPSO退化机制的完整诊断 + OPN4独立测试集的泄露与标签可靠性审计 |
| 跨物种泛化 | 通常不评估 | 通过家族先验与ICL相似度将远缘物种预测准确率从28.6%提升至71.4%（8M）和85.7%（650M） |

## 第七部分：局限性与未来方向

### 7.1 当前局限

| 局限 | 说明 | 影响程度 |
|------|------|----------|
| 缺少3D结构特征 | UniProt注释仅定义loop边界，不编码侧链构象或残基-残基接触 | 高 |
| LOGPSO协议-表征不兼容 | concatenated fixed-vector表征在跨家族评估中必然退化 | 中 |
| 二元家族标签粒度粗 | 仅区分四大G蛋白家族，无法预测亚型（如Gα_q vs Gα_11） | 中 |
| 缺少湿实验验证 | 计算预测未经BRET/Co-IP等实验验证 | 高 |
| 测试标签可靠性 | 大量OPN4测试标签在人源GNAQ框架下未经实验验证（absence of evidence），评估基准本身存在缺陷 | 高 |
| 家族先验过强（8M） | 训练集中opsin家族仅1个Gq正样本（14.3%），8M模型中家族先验将部分远缘正样本压到阈值以下 | 中 |
| Permutation importance样本量 | 50样本子集的统计功效有限 | 低 |
| 负样本定义 | GPCRdb中"non-coupling"属于absence of evidence，可能包含假阴性 | 中 |

### 7.2 未来方向

1. **结构交互表征**：集成AlphaFold预测的界面接触，编码几何依赖的偶联规则
2. **注意力交叉机制**：使用残基级cross-attention或co-embedding替代concatenation，实现跨家族泛化
3. **精细化标签**：从家族级（4类）推进到亚型级预测
4. **实验验证**：对ICL2/3关键残基进行定点突变，通过BRET或Co-IP验证计算预测
5. **扩展物种范围+模型规模协同**：将训练集扩展到无脊椎动物GPCR以平衡家族先验；同时，650M实验表明更大规模的预训练模型能部分缓解稀疏先验对远缘正样本的压制，未来可探索"更大模型×更丰富数据"的协同优化
6. **校准外部测试标签**：对人源GNAQ框架下的远缘物种opsin进行湿实验验证，建立可靠的跨物种评估基准
7. **下一代预训练模型**：测试ESM-2 3B、ESM-3或谱系特异性蛋白语言模型，评估其在跨物种和跨家族泛化上的边际收益

## 第十一部分：一句话总结

**本研究将UniProt精选跨膜注释注入ESM-2预训练embedding，构建真正的(GPCR, G蛋白家族)配对预测框架，并通过引入家族级偶联先验与ICL2/3同源相似度特征，在严格同源性控制下将Cluster CV AUC从0.797提升至0.842（8M）和0.833（650M）；可解释性分析证实模型权重从非功能区向已知结合界面重新分配，而LOGPSO实验揭示了concatenated fixed-vector表征在跨家族泛化中的结构性局限。独立OPN4测试显示：8M V2将干净样本准确率从28.6%提升至71.4%，650M V2进一步跃升至85.7%，但大量无脊椎动物测试标签在人源GNAQ框架下未经实验验证，暴露了当前跨物种评估基准的可靠性缺陷。**

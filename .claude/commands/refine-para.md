# 段落精修命令

## 触发词
/refine

## 期望行为
当我选中一段文本并输入 "/refine" 时，执行以下流程：

1. 读取 `rules/01-academic-tone.md`、`rules/03-argument-framework.md`、`rules/05-no-ai-jargon.md`
2. 对当前段落进行四维诊断：
   - **逻辑维度**：论点-证据-解释链条是否完整
   - **语言维度**：是否有 AI 惯用废话、长难句、冗余修饰
   - **学术维度**：语气是否客观严谨，术语使用是否准确
   - **衔接维度**：与上下文的过渡是否自然
3. 输出格式：
   - **诊断报告**（3-5 条具体问题）
   - **润色后全文**（完整段落，非仅有改动处）
   - **改动说明**（每条改动对应的诊断问题）

## 约束
- 保留原文核心观点，不添加原文未提及的新内容
- 如有必要添加过渡句，用 [新增过渡] 标注

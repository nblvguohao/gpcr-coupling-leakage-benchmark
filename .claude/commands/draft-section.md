# 快速起草章节命令

## 触发词
/draft

## 期望行为
当我输入 "/draft <章节名>" 时，执行以下流程：
1. 读取根目录下的 `CLAUDE.md` 作为核心规则。
2. 读取 `rules/01-academic-tone.md` 作为语气规范。
3. 不生成废话介绍，直接给出：
   - 一段标准的章节框架文本（1000 字左右）。
   - 框架内使用 [待补充] 标记占位符。
   - 在文末附上 3 个具体的问题，以便我后续完善。

## 示例
输入：`/draft 实验方法`

输出：
- 实验方法章节的完整框架
- 包含被试、材料、程序、数据分析等子节
- 每个子节的关键参数处用 [待补充] 占位
- 文末 3 个引导性问题

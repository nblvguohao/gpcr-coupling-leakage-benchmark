# 跨章逻辑一致性检查命令

## 触发词
/logic

## 期望行为
当我输入 "/logic <章A> <章B>" 时，执行以下流程：

1. 同时读取两章的完整内容
2. 执行以下检查：
   - **术语一致性**：两章中使用的核心概念、定义是否前后统一
   - **假设连贯性**：A 章提出的研究假设是否在 B 章得到对应检验
   - **论证闭环**：A 章提出的问题是否在 B 章得到回应
   - **变量对应**：A 章定义的自变量/因变量是否与 B 章的操作化定义匹配
   - **范围一致性**：A 章声称的研究范围是否与 B 章的实际操作一致
3. 输出格式：
   - **一致性确认清单**（逐项 check，标注通过/有问题）
   - **矛盾点详述**（引用两章原文对比，指出具体矛盾）
   - **修复建议**（针对每个矛盾给出至少一种修复方案）

## 示例
输入：`/logic 第二章 第三章`

用途：检查文献综述中提出的理论缺口，是否在研究方法中设计了对应的检验手段。

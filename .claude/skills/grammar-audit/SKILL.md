# 语法审查技能

## 触发方式
- 自动触发：每次保存文档时（通过 settings.json 中的 hooks 配置）
- 手动触发：输入 "/grammar-audit" 或 "检查语法"

## 审查维度
1. **基础语法**
   - 主谓一致
   - 时态统一（文献综述用现在时，方法用过去时，结果用过去时，讨论用现在时）
   - 标点规范（中文用全角，英文用半角）
   - "的/地/得" 使用正确性

2. **学术语法**
   - 避免口语化表达
   - 避免连续使用相同的句式结构
   - 避免头重脚轻的长定语
   - 单句长度不超过 50 字

3. **术语一致性**
   - 同一概念在全文中使用统一术语
   - 首次出现的缩写必须给出全称
   - 翻译术语与学界惯例一致

## 输出格式
- 用表格列出所有问题：位置 | 问题类型 | 原文 | 修改建议
- 问题按严重程度排序：严重 > 中等 > 轻微
- 如果零问题，输出 "✅ 语法检查通过"
